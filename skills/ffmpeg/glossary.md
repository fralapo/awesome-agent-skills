# FFmpeg Glossary

**ABR** — Adaptive bitrate streaming; multiple renditions (HLS/DASH) switched by client.
**AAC** — Advanced Audio Coding; default modern lossy audio (`-c:a aac`, encoder `aac`). (Ch04)
**amix / amerge / pan** — Audio filters: mix multiple inputs / merge into multichannel / remap channels. (Ch06)
**Bitstream filter** — Packet-level transform without full re-encode, e.g. `h264_mp4toannexb`, `-bsf:v`. (Ch04)
**Container / format** — File wrapper (mp4, mkv, mov, ts) handled by muxers/demuxers. (Ch05)
**CRF** — Constant Rate Factor; 1-pass constant-quality control for x264/x265 (lower=better). (Ch04)
**Codec** — Coder/decoder; `-c:v`/`-c:a` select encoder; `copy` = passthrough. (Ch04)
**Demuxer** — Reads a container and separates elementary streams on input. (Ch05)
**drawtext** — Video filter rendering text (needs libfreetype). (Ch06)
**EBU R128 / loudnorm** — Loudness normalization standard / its FFmpeg filter. (Ch06)
**Elementary stream** — A single audio/video/subtitle/data track within a file.
**faststart** — `-movflags +faststart`; relocates MP4 moov atom to front for progressive playback. (Ch05)
**Filtergraph** — Chain/network of filters; `-vf`/`-af` (simple) or `-filter_complex` (multi-io). (Ch06)
**fps / framerate** — Frames per second; `-r` (output rate) vs `-framerate` (input) vs `fps` filter. (Ch03,Ch06)
**framesync** — Shared options for multi-input filters (overlay, etc.): `eof_action`, `shortest`, `repeatlast`. (Ch06)
**GOP / -g** — Group of Pictures; keyframe interval; align to segment size for streaming. (Ch04)
**HLS** — HTTP Live Streaming; `-f hls`, `.m3u8` + segments. (Ch05,Ch07)
**Keyframe (I-frame)** — Independently decodable frame; copy-cuts snap here. (Ch04)
**libx264 / libx265** — H.264 / H.265 (HEVC) software encoders. (Ch04)
**libmp3lame / libopus / libvorbis** — MP3 / Opus / Vorbis audio encoders. (Ch04)
**-map** — Manual stream selection into output; overrides automatic selection. (Ch02)
**-movflags** — MP4/MOV muxer flags (`+faststart`, `+frag_keyframe`). (Ch05)
**Muxer** — Writes elementary streams into a container on output. (Ch05)
**NVENC / QSV / VAAPI / VideoToolbox** — Hardware encoders: NVIDIA / Intel QuickSync / Linux VA-API / Apple. (Ch04,Ch06)
**pix_fmt** — Pixel format/chroma layout (e.g. `yuv420p` for compatibility). (Ch03,Ch09)
**Preset** — Encoder speed/efficiency tradeoff (`ultrafast` to `placebo`). (Ch04)
**Protocol** — Input/output transport: file, pipe, http, tcp, udp, rtmp, rtp, srt, tee. (Ch07)
**-re** — Read input at native frame rate (for streaming files in realtime). (Ch07)
**Remux** — Repackage streams into a new container without re-encoding (`-c copy`). (Ch02)
**scale / crop / pad** — Core geometry video filters. (Ch06)
**-ss / -t / -to** — Seek start / duration / end time; placement (pre/post `-i`) changes behavior. (Ch03)
**Stream copy** — `-c copy`; passthrough, no re-encode. (Ch02)
**Stream specifier** — Selector like `v:0`, `a:1`, `m:language:eng`, `p:1`. (Ch02)
**swscale / swresample** — Libraries for video scaling+pixfmt and audio resampling. (Ch09)
**tee muxer** — Send one encode to multiple outputs simultaneously. (Ch07)
**Timeline editing** — Enable a filter only on a time range via `enable=between(t,a,b)`. (Ch06)
**2-pass** — Two-run encode for accurate target bitrate. (Ch04)
**yuv420p** — 8-bit 4:2:0 planar pixel format; the compatibility baseline. (Ch09)
